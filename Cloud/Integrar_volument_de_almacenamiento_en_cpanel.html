Detén los servicios de cPanel: Antes de mover los datos, es importante detener todos los servicios relacionados con cPanel para evitar cualquier corrupción de datos durante la transferencia. sudo systemctl stop cpanel sudo systemctl stop httpd sudo systemctl stop mysqld 2. Copia los datos de cPanel a la nueva partición: Suponiendo que los datos de cPanel se encuentran en /home, utiliza rsync para copiar todos los datos a la nueva partición montada en /mnt/home: sudo rsync -avz /home/ /mnt/home/ Este comando copiará todos los archivos y carpetas de /home a /mnt/home, preservando los permisos, fechas y otros atributos. 3. Verifica la copia: Es recomendable verificar que todos los datos se hayan copiado correctamente antes de proceder: sudo diff -r /home /mnt/home 4. Actualiza el archivo /etc/fstab: Como ya has añadido la línea para montar la nueva partición en /mnt/home, debes modificar este archivo para que la nueva partición se monte en /home en lugar de /mnt/home. sudo nano /etc/fstab /dev/disk/by-id/scsi-0HC_Volume_101204122 /mnt/home ext4 discard,nofail,defaults 0 0 a /dev/disk/by-id/scsi-0HC_Volume_101204122 /home ext4 discard,nofail,defaults 0 0 5. Renombra la carpeta /home original y monta la nueva partición: Renombra el directorio /home actual por seguridad: sudo mv /home /home.bak 6.Crea un nuevo directorio /home vacío: sudo mkdir /home 7. Monta la nueva partición: sudo mount -a Reinicia los servicios de cPanel: Finalmente, vuelve a iniciar los servicios de cPanel para que todo funcione desde la nueva ubicación: sudo systemctl start cpanel sudo systemctl start httpd sudo systemctl start mysqld Verifica que todo funcione correctamente: Asegúrate de que cPanel esté funcionando correctamente después de la migración.
